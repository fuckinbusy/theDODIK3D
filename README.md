```bat
build.bat
```

Бинарник: `game\x64\Debug\theDODIK3D.exe`

## Changelog

### v0.6 — 2026-03-01

#### Исправлено
- **[assets.c]** Баг в `assets_read_atlas`: вычисление столбца текстуры использовало побитовое `& (cols - 1)` вместо `% cols`.
  Побитовый трюк корректен только когда `cols` — степень двойки. Атлас додика имеет 6 колонок (384 / 64 = 6),
  из-за чего `16 & 5 = 0` вместо правильного `16 % 6 = 4`: текстура `TEXTURE_UI_DODIK_FRAME` (индекс 16)
  загружалась с позиции col=0 row=2, где находится `TEXTURE_UI_DODIK_DEAD`.
  Эта текстура DEAD накладывалась поверх лица додика на каждом кадре, скрывая реальное состояние.
  Исправлено заменой `& (cols - 1)` → `% cols`.
